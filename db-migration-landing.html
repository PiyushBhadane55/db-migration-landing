Features

Everything a real migration needs.

Not just a wrapper around pg_dump. A complete safety harness around the entire migration lifecycle.

Auto Backup + Auto Rollback
Target is snapshotted before every migration. If anything fails, the system restores the backup automatically. No manual intervention required.
Real-Time Log Streaming
Every line from pg_dump and pg_restore is streamed to the browser terminal live. No more staring at a frozen progress bar for 20 minutes.
Disk Space Pre-Check
Checks available disk space before Step 1 starts. A 50GB database silently filling your disk mid-migration is now impossible.
Multi-Schema Support
Validates all PostgreSQL schemas (A1, B1, C1…) — tables, views, constraints, and row counts — per schema. Not just the public schema.
Role Migration (PostgreSQL)
Dumps roles via pg_dumpall --globals-only and restores them to the target. Existing roles are skipped automatically.
Full Validation Layer
Post-restore JDBC check covers tables, views, constraints (PK/UK/FK/CHECK), and row counts with ±5% delta tolerance per table.
Ping / Reachability Check
TCP ping any DB host before migrating. Shows resolved IP, avg/min/max latency, packet loss, and per-probe results. No ICMP or root required.
Persistent Audit History
Every migration persisted to embedded H2 DB with full logs. Browse history, view per-run logs, track durations — survives restarts.
Browser UI, Zero Setup
One JAR, one command — browser opens automatically. No Docker, no npm, no config files. Works on Windows, Linux, and macOS.

Deployment Model

Run it from your laptop.
Reach any server.

No agent on the servers. No SSH tunnels. Just network access to the DB port.

✓ Nothing needs to be installed on Server A or Server B — not even Java. The dump lands on your laptop's disk. Your laptop talks to both servers over TCP port 5432. Works on any LAN, VPN, or cloud environment where the DB port is reachable.

Pipeline

Five steps.
Fully automated.

Every step runs sequentially. Any failure triggers immediate rollback from the Step 1 backup.

01
Backup Target
Snapshot the target DB before any overwrite. Safety net for auto-rollback.
02
Dump Source
pg_dump / mysqldump with live line-by-line output. Disk space checked first.
03
Verify
Dump file integrity check. Size and checksum confirmed before restore.
04
Restore
pg_restore / mysql client on target. Live output. Warnings distinguished from errors.
05
Validate
JDBC checks tables, views, constraints, and row counts per schema. Full report.
